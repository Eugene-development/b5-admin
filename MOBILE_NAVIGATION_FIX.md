# Исправление проблемы с мобильным меню

## Проблема
В мобильном меню при клике на пункты навигации происходила перезагрузка страницы, что приводило к потере состояния авторизации и редиректу на страницу логина. В десктопной версии эта проблема отсутствовала.

## Причина
Мобильное меню использовало обычные HTML ссылки `<a href="...">`, которые вызывают полную перезагрузку страницы. При перезагрузке:
1. Состояние авторизации сбрасывается
2. Происходит повторная инициализация приложения
3. Компонент `ProtectedRoute` перенаправляет неавторизованного пользователя на `/login`

## Решение
Заменили все ссылки `<a href="...">` в мобильном меню на кнопки `<button>` с программной навигацией через `goto()`:

### Изменения в `src/routes/(app)/+layout.svelte`:

1. **Добавлена функция обработки мобильной навигации:**
```javascript
function handleMobileNavigation(path) {
    closeMobileMenu();
    goto(path);
}
```

2. **Заменены все ссылки на кнопки:**
```html
<!-- Было -->
<a href="/dashboard" class={getNavClasses('/dashboard')} onclick={closeMobileMenu}>

<!-- Стало -->
<button type="button" class={getNavClasses('/dashboard')} onclick={() => handleMobileNavigation('/dashboard')}>
```

3. **Обновлены CSS классы для кнопок:**
- Добавлен `w-full text-left` для корректного отображения
- Кнопки теперь выглядят и ведут себя как ссылки

### Затронутые пункты меню:
- Дашборд (`/dashboard`)
- Агенты (`/agents`)
- Кураторы (`/curators`)
- Контрагенты (`/contractors`)
- Поставщики (`/suppliers`)
- Сервис (`/services`)
- Клиенты (`/clients`)
- Проекты (`/projects`)
- Финансы (`/finance`)
- Профиль (`/profile`)
- Настройки (`/settings`)
- Войти (`/login`)

## Результат
- ✅ Мобильное меню больше не вызывает перезагрузку страницы
- ✅ Состояние авторизации сохраняется при навигации
- ✅ Нет редиректов на страницу логина после успешной авторизации
- ✅ Поведение мобильного меню теперь идентично десктопному
- ✅ Улучшена производительность (нет полной перезагрузки)

## Тестирование
1. Авторизуйтесь в системе
2. Откройте мобильное меню
3. Кликните на любой пункт навигации
4. Убедитесь, что:
   - Страница не перезагружается
   - Состояние авторизации сохраняется
   - Навигация работает корректно
   - Мобильное меню закрывается после клика
